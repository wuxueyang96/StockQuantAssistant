使用 Python 构建一个服务端程序，要求暴露 Rest API 以供使用，其包含这些能力：
1. 对于给定的股票代码或者名称，首先判断在当前服务端是否已经存在对应的工作流，如果存在则直接返回，不存在则注册四个新的工作流：
  - 判断股票代码属于 A股、港股还是美股，归属不同的市场需要使用不同的 duckdb 路径（不同的 duckdb 路径表示不同的数据库）；
  - 对于任意的一个股票代码，需要存在四张表，分别对应日线数据、120分钟数据、90分钟数据、60分钟数据，如果表不存在则创建，并获取之前 200 个周期（如果有，比如日线数据就是200天，120分钟数据就是之前 200 个 120 分钟数据）的数据并存储，四张表对应四个工作流；
  - 如果给定的是名称，则通过名称找到股票代码，`市场_股票代码_表名`作为工作量的唯一标识；
  - 四个工作流都需要判断是否是在交易日时间段；
  - 日线数据每天执行一次获取数据的逻辑，并写入到 duckdb 中对应的表里；
  - 120 分钟数据每 120 分钟执行一次获取数据的逻辑，并写入到 duckdb 中对应的表里；
  - 90 分钟数据每 90 分钟执行一次获取数据的逻辑，并写入到 duckdb 中对应的表里；
  - 60 分钟数据每 60 分钟执行一次获取数据的逻辑，并写入到 duckdb 中对应的表里；
  - 每个工作流在执行的时候都需要判断，当前数据是否是新数据，如果不是则直接返回，如果是才写入到 duckdb 中。
  - 工作流需要具备持久化的能力：每次重启自动加载已有的工作量并启动。
